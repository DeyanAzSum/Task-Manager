# Task & Project Manager

Конзолно приложение на C++ за управление на задачи и проекти.

## Компилиране и стартиране

```bash
mkdir build && cd build
cmake ..
cmake --build .
./taskmanager
```

## Имплементирани функционалности

| № | Функционалност |
| 1 | CRUD на задачи |
| 2 | CRUD на проекти |
| 3 | Управление на потребители и роли |
| 4 | Назначаване на задачи към потребители |
| 5 | Проследяване на статус (с история) |

## Класова йерархия

```
WorkItem (abstract)
├── Task    (наследява WorkItem)
└── Project (наследява WorkItem)

User          — самостоятелен клас
TaskManager   — Singleton контролер
```

## Промени спрямо Чекпойнт 1

| Елемент | План | Имплементация | Причина |
|---------|------|---------------|---------|
| `Comment` клас | Включен в дизайна | Пропуснат | Извън обхвата на имплементираните 5 функционалности; лесно добавим в следваща итерация |
| `Notification` клас | Включен в дизайна | Пропуснат | Същото — не е сред избраните функционалности |
| `Report` клас | Включен в дизайна | Пропуснат | Отчетите не са сред имплементираните функционалности |
| `getDetails()` → `printDetails()` | `getDetails(): String` | `printDetails(): void` | В конзолно приложение директният печат е по-прост и достатъчен; няма нужда от String буфер |
| История на статусите | Не е в плана | Добавена чрез `StatusChange` вектор в `Task` | Проследяването на статус е безсмислено без история; добавено като естествено разширение |

## Структура на файловете

```
include/
  Enums.h       — Priority, Status, Role enum класове
  WorkItem.h    — Абстрактен базов клас
  User.h        — Потребител с роля
  Task.h        — Задача (наследява WorkItem)
  Project.h     — Проект (наследява WorkItem, composition с Task)
  TaskManager.h — Singleton контролер
  Menu.h        — CLI меню логика
src/
  main.cpp      — Входна точка
CMakeLists.txt
```
